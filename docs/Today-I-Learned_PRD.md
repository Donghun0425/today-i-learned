# Today-I-Learned MVP PRD

## 🎯 핵심 정보

**목적**: 주니어 개발자가 현대적 웹 개발 워크플로우를 단계별로 학습하고, 각 단계에 필요한 기술 스택을 체계적으로 습득할 수 있는 학습 플랫폼
**사용자**: 웹 개발 워크플로우(기획→개발→테스트→배포)를 체계적으로 익히고 싶은 주니어 프론트엔드/풀스택 개발자

---

## 🚶 사용자 여정

```
1. 랜딩 페이지
   사용자가 서비스에 처음 접속
   ↓ "학습 시작하기" 버튼 클릭

2. 워크플로우 로드맵 페이지
   전체 개발 워크플로우 단계를 한눈에 확인
   ↓ 관심 있는 워크플로우 단계 클릭

3. 기술 스택 상세 페이지
   해당 단계의 기술 스택 학습 콘텐츠 + 코드 예제 확인
   ↓ "학습 완료" 체크 또는 "TIL 작성" 클릭

   [비로그인 상태] → 로그인 페이지로 리디렉션
                    ↓
                    [계정 없음] → 회원가입 페이지 → 로그인 페이지
                    [계정 있음] → 로그인 → 이전 페이지로 복귀
   ↓

   [로그인 상태]
   ├── "학습 완료" 체크 → 진도 저장 → 워크플로우 로드맵 페이지 (진도 반영)
   └── "TIL 작성" 클릭 → TIL 페이지
       ↓
4. TIL 페이지
   오늘 배운 내용을 마크다운으로 기록
   ↓ 작성 완료

5. 나의 학습 대시보드
   전체 학습 진도율, 작성한 TIL 목록 확인
   ↓ 미완료 기술 스택 클릭 → 기술 스택 상세 페이지로 이동
```

## ⚡ 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|------------|
| **F001** | 워크플로우 로드맵 표시 | 개발 워크플로우(기획→환경설정→프론트엔드→백엔드→형상관리→테스트→빌드→배포→CI/CD→모니터링) 단계를 시각적 로드맵으로 표시 | 서비스의 핵심 가치. 학습 경로를 한눈에 파악 | 랜딩 페이지, 워크플로우 로드맵 페이지 |
| **F002** | 기술 스택 학습 콘텐츠 | 각 워크플로우 단계에 포함된 기술 스택의 개념 설명, 왜 필요한지, 핵심 개념, 사용 시나리오를 제공 | 실제 학습이 이루어지는 핵심 콘텐츠 | 기술 스택 상세 페이지 |
| **F003** | TIL 학습 노트 작성 | 학습한 내용을 마크다운 형식으로 기록하고, 관련 기술 스택에 태깅하여 저장 | 학습 내용 정리를 통한 지식 정착 | TIL 페이지 |
| **F004** | 학습 진도 추적 | 각 기술 스택의 학습 완료 여부를 체크하고, 워크플로우 단계별 진도율을 시각적으로 표시 | 학습 동기 부여 및 현재 위치 파악 | 워크플로우 로드맵 페이지, 나의 학습 대시보드 |
| **F005** | 코드 예제 제공 | 각 기술 스택의 실제 코드 예제를 구문 강조(syntax highlighting)와 함께 제공 | 개념 이해를 넘어 실제 코드 확인 필수 | 기술 스택 상세 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|------------|
| **F010** | 기본 인증 | 이메일 기반 회원가입/로그인/로그아웃 | 사용자별 진도 및 TIL 저장을 위한 최소 인증 | 로그인 페이지, 회원가입 페이지 |
| **F011** | 학습 데이터 조회 | 사용자의 학습 진도 요약, 작성한 TIL 목록, 최근 학습 활동을 한 곳에서 조회 | 학습 현황 파악을 위한 필수 데이터 집합 | 나의 학습 대시보드 |

### 3. MVP 이후 기능 (제외)

- 소셜 로그인 (Google, GitHub, Kakao)
- TIL 공개/공유 기능 (다른 사용자의 TIL 열람)
- 코드 실행 플레이그라운드 (브라우저 내 코드 실행)
- 학습 그룹/스터디 기능
- 알림 시스템 (학습 리마인더)
- 프로필 상세 관리 (아바타, 자기소개)
- 검색 기능 (기술 스택/TIL 검색)
- 댓글/질문 기능

### 워크플로우 로드맵 페이지

> **구현 기능:** `F001`, `F004` | **메뉴 위치:** 워크플로우

| 항목 | 내용 |
|------|------|
| **역할** | 전체 개발 워크플로우를 단계별 시각적 로드맵으로 보여주고, 각 단계에 속한 기술 스택 목록과 학습 진도를 표시하는 핵심 탐색 페이지 |
| **진입 경로** | 랜딩 페이지 "학습 시작하기" 클릭, 상단 메뉴 "워크플로우" 클릭 |
| **사용자 행동** | 워크플로우 단계를 탐색하고, 특정 단계를 펼쳐 기술 스택 목록을 확인하고, 관심 있는 기술 스택을 클릭하여 상세 페이지로 이동. 로그인 시 학습 완료 체크 가능 |
| **주요 기능** | • 10단계 워크플로우 시각적 로드맵 (세로 타임라인 형태)<br>• 각 단계 클릭 시 포함된 기술 스택 목록 펼쳐보기<br>• 로그인 사용자: 각 기술 스택의 학습 완료/미완료 상태 표시<br>• 로그인 사용자: 단계별 진도율 표시 (예: 3/5 완료)<br>• 기술 스택 클릭 → **기술 스택 상세 페이지**로 이동 |
| **다음 이동** | 기술 스택 클릭 → 기술 스택 상세 페이지 |

### 기술 스택 상세 페이지

> **구현 기능:** `F002`, `F005` | **인증:** 열람은 비로그인 가능, 학습완료/TIL 작성은 로그인 필요

| 항목 | 내용 |
|------|------|
| **역할** | 특정 기술 스택의 개념 설명, 핵심 개념, 사용 사례, 코드 예제를 제공하는 학습 콘텐츠 페이지 |
| **진입 경로** | 워크플로우 로드맵 페이지에서 기술 스택 클릭 |
| **사용자 행동** | 기술 스택의 개념 설명을 읽고, 코드 예제를 확인하고, 학습 완료 체크를 하거나 TIL 작성 페이지로 이동 |
| **주요 기능** | • 기술 스택 제목, 속한 워크플로우 단계 표시 (브레드크럼)<br>• 한 줄 요약 + 비유를 사용한 쉬운 설명<br>• 핵심 개념 3~5개 (각 개념별 설명)<br>• 구문 강조가 적용된 코드 예제 블록 (복사 버튼 포함)<br>• 공식 문서 링크 및 참고 자료<br>• **"학습 완료"** 체크 버튼 (로그인 필요)<br>• **"TIL 작성하기"** 버튼 (로그인 필요) |
| **다음 이동** | "학습 완료" → 워크플로우 로드맵 페이지(진도 갱신), "TIL 작성하기" → TIL 페이지(기술 스택 자동 태깅) |

## 🗄️ 데이터 모델

|------|------|----------|

| title | 단계 이름 (예: "프론트엔드 개발") | String |
| description | 단계 설명 | String |
